Add `BankaiCodeBlock` — a block (fenced) code primitive with a copy button. It renders a native `.bankai-code-block` `<div data-part="root">` wrapping a `<pre data-part="pre"><code data-part="code">`, so the block-code semantics live on the elements themselves. The `code` prop is the source of truth: it renders verbatim as the code body (or pass pre-highlighted markup through the default slot) and is the exact string the copy button writes to the clipboard. Core highlights nothing (design-agnostic) — the optional `language` prop only reflects as the de-facto `language-<lang>` class on the `<code>` for a BYO highlighter (Shiki, Prism, highlight.js) or CSS to key off. The copy button is a composed `BankaiButton` whose accessible name tracks its visible label (WCAG 2.5.3); a successful copy reflects a transient `data-bankai-copied` state and is announced through a visually hidden `role="status"` live region (re-announced on a repeat copy). Localize both the button text and the announcement via the `copyLabel` / `copiedLabel` props, customize the button content via the `copy` slot (which receives the `copied` state), or drop the button with `:copyable="false"`. The copied state's duration is tunable per-instance via `copiedDuration` (ms) and globally via the new `BankaiConfig.codeBlockCopiedDuration` (default `2000`). Distinct from the inline `BankaiCode`. Core ships no CSS; consumer `class`/`style`/attributes merge onto the root, and the owned `data-part`/`data-bankai-*` can't be clobbered by fallthrough.
