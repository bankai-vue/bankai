Add `BankaiIcon` — an agnostic icon wrapper that provides a consistent 1:1 box, sizing, `currentColor` alignment, and accessibility around whatever icon mechanism the consumer already uses (an inline `<svg>` or icon component, a CSS icon font, or a mask-based set like UnoCSS / Iconify-CSS). Core ships **no** icons. Three input styles: the default slot (any markup), a CSS icon `class` straight through, or a `name` token — the same as `class` but routed through an optional `config.iconClass(name)` resolver (unset → the token is applied verbatim, so UnoCSS / Iconify-CSS `name="i-mdi-home"` works out of the box). `size` is a named `xs`–`xl` step (reflected as `data-bankai-size`, mapped to `--bankai-icon-size-*`) plus a verbatim escape hatch (any CSS length on `--bankai-icon-size`); one `font-size` knob scales inline SVG, masks, and glyph fonts alike. The box is square by default and non-square content is centered and letterboxed rather than distorted; opt out with `no-square` (reflected as `data-bankai-square="false"`). Icons are **decorative by default** (`aria-hidden`); pass `label` for a meaningful icon (`role="img"` + `aria-label`), and an explicit consumer `aria-label`/`aria-labelledby`/`title`/`role` is respected rather than overridden. Polymorphic `as` (default `<span>`, `'i'` suggested). Adds an optional `iconClass` field to the global config. Core ships no CSS.
