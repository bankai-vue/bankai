Add `BankaiText` — a polymorphic (`as`, default `<span>`) text primitive with typed `size`, `weight`, `tone`, and `truncate` props. Each styling prop is a first-class named set plus an escape hatch: `size` takes a `xs`–`2xl` t-shirt step or any CSS length; `weight` takes a named step (`thin`–`black`), a `number`, or any CSS string (`var()`/`calc()`) for the variable-font `wght` axis; `tone` takes a neutral tone (`default`/`muted`/`subtle`) or any CSS color; `as` suggests the non-interactive phrasing/text elements (`BankaiTextElement`) but accepts any tag string (custom elements). Named values reflect as a `data-*` anatomy (`data-bankai-size`/`data-bankai-weight`/`data-bankai-tone`/`data-bankai-truncate`) the theme maps; verbatim values ride `--bankai-text-*` custom properties applied by a zero-specificity base rule, so they still lose to a consumer's utility class. Inline text semantics (`<strong>`, `<em>`, `<mark>`, `<code>`, `<kbd>`, `<del>`, …) are reached through `as`. Core ships no CSS. Semantic status tones (`success`/`warning`/`danger`/`info`) are deferred to a follow-up color-tokening change (arbitrary colors already work via the `tone` escape hatch).
