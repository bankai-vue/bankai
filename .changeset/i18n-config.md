Add an i18n surface for localizing components' default UI strings, designed once for every component rather than per component (SPEC §4.20). `BankaiConfig` gains a nested `i18n` object — `{ locale, fallbackLocale, messages }` (`BankaiI18nConfig`) — and `createBankai` deep-merges it, so a partial `i18n: { locale: 'de' }` keeps the `fallbackLocale`/`messages` defaults. A new `BankaiMessages` interface is the typed, per-component namespaced registry every locale bundle conforms to (first namespace: `codeBlock.copy` / `codeBlock.copied`), and a `useBankaiMessage()` composable resolves the active, complete message set reactively. English is the always-present, complete base, so a registered bundle may be partial (omitted keys fall through to English, never to an empty string), and unregistered locales stay out of the build. Locale bundles are tree-shakeable opt-in exports — `import { de } from '@bankai-vue/core/locales'` (or `@bankai-vue/core/locales/de`) — registered under `i18n.messages`; a German (`de`) bundle ships. Resolution walks the active `locale` and its regional parents (`de-AT` → `de`) → `fallbackLocale` and its parents → the English base. `BankaiCodeBlock` is the first consumer: its `copyLabel` / `copiedLabel` (and the `role="status"` announcement) now resolve through the config with precedence **prop → active locale bundle → English default**, so the per-instance props still win for a single block while `i18n` localizes the default app-wide. Config is per-app (SSR-safe) and reactive, so switching `locale` at runtime re-renders every label.
