Finalize `BankaiButton` styling — add interaction feedback and a full `--bankai-button-*` override surface. All three variants now respond to `:hover`/`:active` (guarded by `:not(:disabled)`): the `solid` fill shifts via `--bankai-button-bg-hover`/`--bankai-button-bg-active` (darkened toward black, visible in both schemes — a dark button deepens toward black, a light button to a mid gray), and the quiet `outline`/`ghost` variants gain a `currentcolor` tint via `--bankai-button-bg-hover-quiet`/`--bankai-button-bg-active-quiet`.

Every adjustable property now routes through a `--bankai-button-*` component token declared on `:where(:root)`, each defaulting to a shared-scale value (`--bankai-space-*`, `--bankai-radius`, `--bankai-text-size-*`, `--bankai-focus-ring`) or a plain literal — so a consumer restyles by overriding one custom property (e.g. `--bankai-button-display`, `--bankai-button-gap`, `--bankai-button-radius`, `--bankai-button-border-width`/`-style`/`-color`, `--bankai-button-align`, `--bankai-button-font`, `--bankai-button-cursor`, `--bankai-button-transition`, `--bankai-button-focus-ring`/`-focus-outline-width`/`-offset`, `--bankai-button-disabled-opacity`/`-cursor`, and per-size metrics `--bankai-button-{sm,md,lg}-padding-y`/`-padding-x`/`-font-size`) without writing a selector, while a shared-scale retune still flows through the defaults. The color-only `transition` is unaffected by reduced-motion. The border-color token was renamed `--bankai-button-border` → `--bankai-button-border-color` now that `-border-width`/`-border-style` sit beside it.
