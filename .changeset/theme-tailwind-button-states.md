Finalize `BankaiButton` styling in the Tailwind theme — a conventional indigo primary button (indigo fill, white semibold label) with interaction feedback. The `solid` variant is now `font-semibold` with a `text-sm` label on the indigo fill; the `sm`/`md`/`lg` sizes differ only in padding + radius (`sm` → `rounded-sm px-2 py-1`, `md` → `px-2.5 py-1.5`, `lg` → `px-3 py-2`), all sharing the base `text-sm` (previously `lg` used `text-lg`). All three variants respond to `:hover`/`:active` (guarded by `:not(:disabled)`): the `solid` fill steps one lighter along the indigo ramp on hover (`indigo-600`→`500` light, `indigo-500`→`400` dark) and one darker on active for a pressed feel, via `--bankai-button-bg-hover`/`--bankai-button-bg-active` as `light-dark()` pairs keyed off `color-scheme` (no `dark:`); the quiet `outline`/`ghost` variants gain a `currentcolor` tint via `--bankai-button-bg-hover-quiet`/`--bankai-button-bg-active-quiet`. The transition uses Tailwind's `transition-colors` utility; `font-family: inherit` keeps the consumer's font without a shorthand resetting the applied weight/size. The border-color token was renamed `--bankai-button-border` → `--bankai-button-border-color` to stay identical to the house theme's token surface. (`xs`/`xl` sizes are a possible later addition.)
