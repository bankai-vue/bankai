Add the foundation semantic **color tokens** — the discover-as-you-go token system's color slice. `theme-bankai` exposes `--bankai-color-*` roles as `light-dark()` pairs (authored in OKLCH) so they follow the system `color-scheme` with no JS and no flash: `fg`/`fg-muted`/`fg-subtle` (foreground emphasis), `bg`/`surface` (page vs. raised; `bg` is neutral), `border`, `primary`/`primary-fg` (the primary action fill and text on it), and `accent` (interactive accent; the default source of `--bankai-focus-ring`). Components now reference these roles instead of raw colors — `BankaiButton`'s solid variant draws from `--bankai-color-primary`, and `BankaiText`'s neutral tones alias the `--bankai-color-fg*` family — so text and button color resolve from one source of truth (non-breaking: values are unchanged). `theme-tailwind` exposes the same **roles** but an independent **palette**: it mimics the stock Tailwind look, pointing each role at the consumer's Tailwind `--color-*` with `gray` for neutrals and `indigo` for primary + focus accent (e.g. `--bankai-color-primary` → `light-dark(var(--color-indigo-600), var(--color-indigo-500))`), and refactors its button off raw `@apply bg-slate-*`/`outline-blue-*` onto the tokens. Semantic status colors (`success`/`warning`/`danger`/`info`) are a follow-up slice.
