Add `BankaiLink` — a router-aware link primitive that renders the right element for its environment with no consumer wiring: a globally-registered `NuxtLink` (preferred, under Nuxt) or `RouterLink` (vue-router) for internal `to` navigation, and a native `<a>` when there is no router or the target is external. Resolution order is `NuxtLink` → `RouterLink` → `<a>`, detected from the app's global component registry, so `core` keeps its near-zero-dep posture and never imports vue-router at runtime (SPEC §5.5). Props: `to` (internal target; a string degrades to `<a href>` without a router), `href` (always a plain `<a>` — external URLs, `mailto:`/`tel:`, fragments, downloads; wins over `to`), and `external` (force a full-page `<a>`). A `target="_blank"` link gains `rel="noopener noreferrer"` automatically (opt out via `config.linkNoopener`); a consumer `rel` always wins. The root exposes a `.bankai-link` class, `data-part="root"`, and a `data-bankai-external` hook; core ships no CSS and merges consumer `class`/`style`/attributes onto the root. Two new `BankaiConfig` fields — `linkComponent` (override the internal-link component) and `linkNoopener`. For first-class typing of `to` as vue-router's `RouteLocationRaw`, add the optional `@bankai-vue/core/vue-router` types augmentation to your tsconfig `types`; without it (and with no vue-router installed) `to` is a router-agnostic fallback and the package still type-checks under `skipLibCheck: false`. `vue-router` is an optional `peerDependency`.
