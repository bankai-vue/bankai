Add `BankaiLink` — a router-aware link primitive that renders the right element for its environment with no consumer wiring: a globally-registered `NuxtLink` (preferred, under Nuxt) or `RouterLink` (vue-router) for internal `to` navigation, and a native `<a>` when there is no router or the target is external. Resolution order is `NuxtLink` → `RouterLink` → `<a>`, detected from the app's global component registry, so `core` keeps its near-zero-dep posture and never imports vue-router at runtime (SPEC §5.5). Props: `to` (internal target; a string degrades to `<a href>` without a router), `href` (always a plain `<a>` — external URLs, `mailto:`/`tel:`, fragments, downloads), and `external` (force a full-page `<a>`). `to` and `href` are mutually exclusive — a type error if both are set (discriminated union), rather than silently dropping `to`. A `target="_blank"` link gains `rel="noopener noreferrer"` automatically (opt out via `config.linkNoopener`); a consumer `rel` always wins. The root exposes a `.bankai-link` class, `data-part="root"`, and a `data-bankai-external` hook — reflected for a forced `external`, a `target="_blank"` target, or an `href` whose host differs from the current site; core ships no CSS and merges consumer `class`/`style`/attributes onto the root. Host comparison uses `config.linkOrigin` when set (identical on server and client, so it is SSR/SSG hydration-safe), else `window.location` in the browser, else treats any absolute URL as external. New `BankaiConfig` fields — `linkComponent` (override the internal-link component), `linkOrigin` (own site origin for external detection), `linkNoopener`, and `warnings` (global opt-out for development warnings, e.g. an object `to` with no router to resolve it). Warnings are DEV-only (stripped from production builds). For first-class typing of `to` as vue-router's `RouteLocationRaw`, add the optional `@bankai-vue/core/vue-router` types augmentation to your tsconfig `types`; without it (and with no vue-router installed) `to` is a router-agnostic fallback and the package still type-checks under `skipLibCheck: false`. `vue-router` is an optional `peerDependency`.
